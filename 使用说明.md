# soyplant 包使用说明

**soyplant** 是一个大豆育种数据管理 R 包，用于管理从杂交组合到田间种植的全流程数据。

---

## 1. 安装与加载

```r
# 加载所有源文件
source("R/config.R")        # 定义 soy_traits 等全局数据
source("R/info_id.R")
source("R/combination.R")
source("R/population.R")
source("R/plant.R")
source("R/line.R")
source("R/primary_productivity.R")
source("R/advanced_productivity.R")
source("R/advance_breeding_stage.R")
source("R/planting.R")
source("R/design_plot.r")
source("R/utils.R")
source("R/baiaoyun.R")
source("R/yield.R")
source("R/traits.R")
source("R/sqlite.R")

# 全局变量 field 和 soy_traits 必须在使用前加载
field <- read.table("data/field.txt", header = TRUE)
```

---

## 2. 育种流程总览

```
杂交组合(mapa) → 群体(population) → 单株(plant) → 株行(line)
  → 初级产比(primary) → 高级产比(advanced) → 多点鉴定 → 田间种植(planting)
```

每个阶段的核心操作：
- **晋级**：将符合条件（`next_stage` 指向下一阶段）的记录筛选出来
- **世代更新**：名称追加世代号（如 `A:F1` → `A:F2`），`f` 字段 +1
- **ID 生成**：每条记录获得全局唯一 `id` 和 `fieldid`
- **溯源**：记录 `source`、`former_fieldid`、`former_stageid`

## 3. 核心函数详解

### 3.1 杂交组合

#### `combination(ma, pa, memo)`
配制杂交组合，返回母本 × 父本的全部组合。

```r
combi <- combination(ma = c("冀豆12", "冀豆17"),
                     pa = c("中联豆6001", "中联豆6024"),
                     memo = "转基因试验")
# 返回: data.frame(ma, pa, mapa, memo)
```

#### `combination_from_dataframe(mydata, only = TRUE, order = FALSE)`
从文件数据框配组合，`mydata` 必须包含三列：母本、父本、备注。

```r
combi <- combination_from_dataframe(read.xlsx("杂交表.xlsx"))
```

#### `get_combination(mydata, prefix = "ZJ", startN = 1)`
将组合数据框升级为完整的杂交阶段记录（带 id、fieldid、stageid 等字段）。

```r
my_combi <- get_combination(mydata, prefix = "ZJ", startN = 1)
```

#### `combination_matrix(my_combi)`
将组合表转换为母本 × 父本矩阵视图。

---

### 3.2 群体

#### `get_population(my_combi, start_num = 1)`
从杂交组合或群体晋级为群体。

```r
my_pop <- get_population(my_combi, start_num = 1)
# 自动过滤 next_stage == "群体" 的记录
# 世代号追加: name末尾自动加1（F0→F1）
# 新 next_stage = "群体"
```

---

### 3.3 单株

#### `get_plant(my_pop, start_num = 1)`
从群体中升级单株。

```r
my_pop <- data.frame(
  name = c("A", "B"), Sele = c(2, 3),  # Sele = 选择单株数
  f = c(1, 1), path = c("A", "B"),
  fieldid = c("f1", "f2"), stageid = c("s1", "s2"),
  process = c("p1", "p2")
)
names(my_pop)[2] <- "sele"

my_plant <- get_plant(my_pop)
# A (选择2株) → A-1, A-2
# B (选择3株) → B-1, B-2, B-3
# 新 stage = "单株", next_stage = "株行"
```

**注意**：`sele` 字段必须为数值型（纯数字），非数字内容会被自动过滤为 NA。

```r
my_line <- get_line(my_plant, start_num = 1)

```

```r
my_primary <- get_primary(my_line, start_num = 1)

```

### 3.7 田间种植设计

#### `planting(my_primary, ck, interval, s_prefix, rp, treatment, place, ckfixed, digits, rows, restartfid, startN)`

完整种植方案设计，自动完成：插入对照 → 添加重复 → 添加处理 → 添加地点 → 编号。

```r
# 基本用法
planted <- planting(
  my_primary = my_primary,      # 来自 get_primary 的数据
  ck = c("冀豆12", "冀豆17"),  # 对照名称
  interval = 3,                # 每隔几个材料插一个对照
  s_prefix = "GC",            # 田间编号前缀
  rp = 2,                      # 重复数（第一重复顺序，其余随机）
  treatment = c(""),           # 处理因素（留空或多个）
  place = c("石家庄", "德州"), # 试验地点
  ckfixed = TRUE,              # TRUE=固定插入对照 FALSE=随机对照
  digits = 3,                  # 田间编号位数
  rows = 6,                    # 每块行数
  restartfid = FALSE,          # 不同地点是否重新编号
  startN = 1
)
```

**田间编号生成规则**（`generate_stageid`）：
- 格式：`前缀 + 年份(2位) + 月(base62) + 日(base62) + 时(base62) + 分(base62) + 秒(base62) + 序号`
- 例：`GC260408143012001` = GC + 26年 + 04月 + 08日 + 14时 + 30分 + 12秒 + 001

**对照插入策略**：
- `ckfixed = TRUE`（固定）：等间隔插入对照，每重复内对照位置相同
- `ckfixed = FALSE`（随机）：对照位置每重复随机，且相邻重复并排两列材料不相同

**地点编号策略**：
- `restartfid = TRUE`：每个地点重新从 1 开始编号（依赖时间戳间隔 1.1 秒保证唯一性）
- `restartfid = FALSE`：全局连续编号

#### `rows_to_linenumber(number)`
将每块行数转换为行号字符串。

```r
rows_to_linenumber(c(3, 2, 4))
# 返回: c("1-3", "4-5", "6-9")
```

#### `addrpckfixed(my_primary, ck, interval, s_prefix, rp, digits, startN)`
固定对照的田间重复设计（内部函数）。

#### `addrpck(my_primary, ck, interval, s_prefix, rp, digits, startN)`
随机对照的田间重复设计（内部函数）。

### 3.8 田间小区设计

#### `expand_rows(df, replicate_col, sequence_col)`
按某列数值展开数据框行。

```r
df <- data.frame(id = c(1, 2, 3), name = c("A", "B", "C"), rows = c(2, 3, 1))
expand_rows(df, "rows", "seq_id")
# 返回 6 行: A(×2), B(×3), C(×1)，附 seq_id 序号
```

#### `getWaterLaneProtected(blockStructure)`

```r
getWaterLaneProtected("w/23/2r/w/2p/w/2p/23/w")
# 返回: totalColumns, waterColumns, laneColumns, protectedColumns
```

生成随机区组设计图。


## 4. 全局字段对齐

所有晋级函数末尾调用 `align_to_field_schema(df, table_pattern)`，确保：
1. 数据框包含该阶段所需的全部字段（缺失则补 NA）
2. 字段顺序与 `field` 表定义一致

必需字段顺序示例（combination 阶段）：
```
id, user, stageid, name, ma, pa, mapa, memo, stage, next_stage,
f, Sele, process, path, source, fieldid, former_fieldid, former_stageid
```

如需在数据框中强制对齐字段：

```r
field <<- read.table("data/field.txt", header = TRUE)  # 必须先加载全局 field
df <- ensure_and_reorder_columns(df, required_columns)
```

## 5. 完整工作流示例

```r
# 1. 加载
source("R/config.R")      # 定义 soy_traits
source("R/info_id.R")     # ID/编号生成（get_computer_nodename, generate_id 等）
source("R/combination.R")
source("R/population.R")
source("R/plant.R")
source("R/line.R")
source("R/primary_productivity.R")
source("R/planting.R")
source("R/utils.R")

field <- read.table("data/field.txt", header = TRUE)

# 2. 配制杂交组合
mydata <- data.frame(ma = c("冀豆12","冀豆17"), pa = c("中联豆6001"), memo = "试验")
combi <- combination_from_dataframe(mydata)
my_combi <- get_combination(mydata, prefix = "ZJ", startN = 1)

# 3. 晋级群体
my_pop <- get_population(my_combi)

# 4. 升级单株（假设已选择）
my_plant <- get_plant(my_pop)

# 5. 晋级株行
my_line <- get_line(my_plant)

# 6. 晋级初级产比
my_primary <- get_primary(my_line)

# 7. 田间种植设计
planted <- planting(my_primary,
                    ck = c("冀豆12"),
                    interval = 3,
                    rp = 2,
                    place = c("石家庄"))

# 8. 导出到 Excel
library(openxlsx)
wb <- createWorkbook()
addWorksheet(wb, "种植")
writeData(wb, "种植", planted)
saveWorkbook(wb, "种植设计.xlsx", overwrite = TRUE)
```

## 7. 注意事项

1. **全局变量 `field`**：大部分函数依赖 `field` 数据框定义字段结构，使用前必须加载 `field <- read.table("data/field.txt", header = TRUE)`。

2. **`sele` 字段**：`get_plant` 中的 `sele` 必须为纯数字，非数字内容（字母、混合字符串）会被过滤为 NA。

3. **ID 唯一性**：`generate_id` 依赖时间戳，同一秒内生成的 ID 严格递增，不需要 `Sys.sleep()`。

4. **代数上限**：`increment_generation_name` 默认最大代数 9，超过不追加。

5. **SQLite 并发**：同一数据库连接不可同时读写，建议函数级连接（`on.exit` 保证关闭）。
